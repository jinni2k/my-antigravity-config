# oh-my-agent 전역 설정 동기화 저장소

이 저장소는 안티그래비티 IDE 및 `oh-my-agent`의 핵심 환경 설정(MCP 서버 설정, 전역 동작 옵션, 커스텀 규칙, 워크플로우 템플릿 등)을 깃허브(GitHub)를 통해 여러 컴퓨터 간에 공유하고 안전하게 동기화하기 위한 용도입니다.

## 동기화 대상 목록

* `oma-config.yaml` : 전역 언어 설정(ko), 모델 프리셋 설정 등
* `mcp.json` : Serena MCP 서버 설정
* `rules/` : 안티그래비티 AI가 준수해야 할 전역 행동 규칙 지침
* `workflows/` : 병렬 에이전트 오케스트레이션 및 코드 리뷰 등의 행동 템플릿
* `sync.ps1` : 원클릭 백업 및 복원 통합 스크립트

*(참고: `skills/` 폴더 하위의 대용량 스킬 소스 파일들은 동기화 용량 축소를 위해 제외되며, 복원 단계 시 자동으로 재다운로드됩니다.)*

---

## 사용 방법

모든 백업과 복원 절차는 단 하나의 스크립트인 `sync.ps1`로 간소화되었습니다.

### 1. 설정 백업 및 동기화 (GitHub Push)

설정이 변경되었거나 새로운 커스텀 규칙을 등록했을 때, GitHub에 업로드합니다.
```powershell
cd ~\.agents
.\sync.ps1
# 메뉴창이 뜨면 [1]번을 선택합니다. (혹은 매개변수로 .\sync.ps1 -Push 실행 가능)
```

### 2. 다른 컴퓨터에서 설정 복원 (GitHub Pull & 스킬 복구)

새로운 컴퓨터에 동일한 설정을 복구하는 절차입니다.

#### A. 깃허브 저장소 클론 (최초 1회)
새 컴퓨터의 사용자 홈 디렉토리 하위에 `.agents` 폴더 이름으로 이 저장소를 클론합니다.
```powershell
cd ~
git clone https://github.com/jinni2k/oh-my-agent-config .agents
```

#### B. 설정 복원 및 스킬 다운로드
클론이 완료되면 `.agents` 폴더로 진입하여 복원 프로세스를 작동시킵니다.
```powershell
cd ~\.agents
.\sync.ps1
# 메뉴창이 뜨면 [2]번을 선택합니다. (혹은 매개변수로 .\sync.ps1 -Pull 실행 가능)
```
* 복원 과정에서 `oh-my-agent doctor`가 자동으로 구동되어, 제외했던 32개 스킬 팩들을 레포지토리로부터 깨끗한 상태로 자동 복구합니다.

#### C. 개발 프로젝트 연동
새로운 개발 프로젝트 폴더를 열고, 프로젝트 터미널에서 다음 명령을 실행하여 로컬 동기화를 마칩니다.
```powershell
oh-my-agent link
```
이 과정을 거치면 새 컴퓨터에서도 이전 컴퓨터와 100% 동일한 안티그래비티 개발 규칙(한글 지침 등)이 활성화되어 바로 동작합니다.
